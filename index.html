Title: Lepton, a robust, high performance parallel image processing library leveraging SIMD and GPU processing through iOS frameworks Metal and Accelerate

Summary: We are building a high-performance image processing library on iOS that uses SIMD and GPU-based processing through Metal and Accelerate Frameworks

Background: With the pervasiveness of social media applications such as Facebook and Snapchat, mobile phones have become the most ubiquitous computing devices in the world. One of the most common tasks these devices are used for is capturing and modifying images and videos in real time. However, massively parallel operations like image processing and live filters for videos are better accomplished on GPUs than on CPUs and the currently available libraries for mobile devices do not take advantage of the full computing power of iPhones and so we decided to see how much we can push parallelism within the resource constraints of these devices.

We are building a framework for image processing that will be robust and abstracted well enough that it is easy to use, on top of being highly performant by taking full advantage of Accelerate and Metal. With our framework iOS developers can have access to a faster image processing library in developing social media applications.

Challenges: This is a significant task with a variety of challenges we have to surpass Designing the APIs and the architecture of the framework taking into consideration the use cases and the limited time we have while managing our workload is a serious undertaking. Because this project is limited in scope to around a month, we should only focus on demonstrating a few image processing routines and not attempt to be as complete as GPUImage We should figure out how to parallelize different tasks and how much GPU resources vs how much CPU resources we ought to allocate We have to come up with a way of scheduling tasks Despite all of this we should always bear in mind the resource limitations of iOS hardware. Learning new tools and domains, neither of us are very experienced with Metal or Accelerate and so learning the platforms would be a challenge If we reach our stretch goal, distributing image processing dynamically in a mesh network where each node is performing parallel computation heavy tasks is an unprecedented task.

Resources: https://github.com/BradLarson/GPUImage GPUImage is a framework that uses OpenGL E.S and CoreImage to get GPU-accelerated performance. While the state of technology in iOS has significantly advanced, GPUImage gives us a starting point for understanding how an image processing framework should work and what features it should contain.

Developer.apple.com contains docs for Accelerate and Metal.
